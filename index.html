Metric 01Network Degree · Local

연결 정도

바깥 노드를 클릭해 연결 토글degree = 4
바깥 노드를 클릭하면 중심 노드(나)와의 연결이 추가/제거된다.

개인이 직접 연결된 사람의 수를 세는 가장 기본적인 지표다.

내가 바로 아는 사람이 몇 명인지를 측정한다. 값이 10이면 나는 10명과 직접 연결되어 있다. 단순하지만 "인맥의 양"을 보여주는 직관적인 지표다.

degree(v) = 노드 v에 연결된 엣지의 개수
Metric 02Betweenness · Bridge

매개 중심성

노드를 클릭하면 매개 중심성 값 표시노드를 선택하라
노드를 클릭하면 그 사람이 지나는 최단 경로들이 강조되고, 매개 중심성 값이 표시된다. 두 그룹을 잇는 다리 노드의 값이 가장 크다.

서로 모르는 두 사람을 연결하는 '다리(bridge)' 역할의 정도다.

전체 네트워크에서 임의의 두 사람을 잇는 최단 경로 중, 나를 거쳐 가는 비율을 모두 합한 값이다. 값이 높을수록 정보·자원이 흐르는 핵심 통로이며, 내가 빠지면 단절되는 관계가 많아진다. 조직의 '브로커' 또는 '게이트키퍼'다. 전역(global) 지표로, 네트워크 전체를 알아야 계산된다.

betweenness(v) = Σ (v를 지나는 최단경로 / 전체 최단경로)
Metric 03Closeness · Reach Speed

근접 중심성

노드를 클릭해 전파 단계 확인노드를 선택하라
색이 진할수록 가깝고(1단계), 옅을수록 멀리(2~3단계) 떨어져 있다.

네트워크 내 모든 사람에게 얼마나 빨리 도달할 수 있는지를 나타낸다.

정보를 전달할 때 그룹 내 모든 사람에게 전파되는 속도다. 다른 모든 노드까지의 최단 경로 거리 합이 작을수록 근접 중심성이 높다.

closeness(v) = (N−1) / Σ 최단거리(v, 모든 노드)
Metric 04Clustering · Closure

클러스터링 계수

이웃들 사이 점선을 클릭해 연결/해제CC = 0.33
왼쪽 녹색 노드가 '나'다. 오른쪽 이웃 4명 사이의 점선을 클릭해 연결해 보라. 이웃끼리 많이 연결될수록 계수가 1에 가까워진다.

내 지인들끼리 서로 얼마나 잘 아는지를 나타내는 값(0~1)이다.

나(녹색 노드)는 옆에 두고, 나와 연결된 이웃들끼리의 연결만 본다. 이웃이 4명이면 이웃 간 가능한 연결은 6쌍이다. 이 중 실제로 몇 쌍이 연결되어 있는지가 계수다. 모두 연결되면 c=1, 절반이면 c=0.5, 하나도 없으면 c=0. 값이 높을수록 촘촘한 소집단(클리크)이라 신뢰가 높지만 외부 정보 유입은 제한된다.

CC = 이웃들 간 실제 연결 수 / 가능한 모든 연결 수 [n(n−1)/2]
Metric 05Density · Team-level

네트워크 밀도

슬라이더로 밀도 조절density = 0.30
희소촘촘
슬라이더를 오른쪽으로 옮길수록 더 많은 연결이 추가된다.

전체 네트워크가 얼마나 촘촘하게 연결되어 있는지를 0~1로 표현한다.

가능한 모든 연결 중 실제로 연결된 비율이다. 밀도가 높으면 신뢰·협력이 강하지만 새 정보가 적고, 낮으면 다양한 정보에 접근하기 좋지만 응집력이 약하다.

density = 실제 엣지 수 / [N(N−1)/2]
Metric 06Structural Hole · Burt 1992

구조적 공백

제약 = 낮음
녹색 '나'는 서로 분리된 4개 그룹에 연결되어 구조적 공백을 가로지르는 위치다(제약 최저). 두 그룹을 잇는 점선을 클릭해 연결하면 — 그 두 그룹이 더 이상 독립된 정보원이 아니게 되어(공백이 메워짐) 나의 제약이 올라가고 유효 규모가 줄어든다. 모든 그룹이 서로 연결되면 나는 더 이상 중개자가 아니다. 클릭하는 빨간 선이 바로 '메워진 공백'이다.

내 이웃들이 서로 얼마나 안 겹치는지를 본다. 서로 모르는 이웃들 사이의 '빈틈(공백)'을 메우는 사람이 정보·기회·통제력의 이점을 얻는다.

이웃들이 서로 겹치지 않을수록 이점이 크다. 유효 규모(Effective size) = 중복을 걷어낸 실질 인맥 수(클수록 좋음) · 효율성(Efficiency) = 인맥 1명당 이점(1에 가까울수록 좋음) · 제약(Constraint) = 공백을 못 누리는 정도(낮을수록 좋음).
제약 cᵢ = Σⱼ ( pᵢⱼ + Σ_q pᵢ_q · p_qⱼ )²
pᵢⱼ = i가 j에 투자한 관계 비중 · q = i,j 모두와 연결된 제3자
