Intro운과 네트워크

'운'이 좋은 사람은 누구일까?

운이 좋다는 것은 확률적으로 자신에게 필요한 지식·정보·힘을 가진 사람과 만나는 것을 의미합니다. 그 '만남의 확률'은 우연이 아니라, 내가 어떤 네트워크 구조 안에 위치해 있는지에 따라 결정됩니다.

사회적 자본(Social Capital)은 바로 이 관계망이 만들어내는 자원입니다.

왼쪽 메뉴에서 7가지 핵심 지표를 차례로 살펴봅니다. 각 시각화의 노드·점선·슬라이더를 직접 만져보고, 마지막 🧪 나의 네트워크 측정에서 내 인맥으로 같은 지표를 계산해 보세요.
Metric 01Network Degree · Local

연결 정도

개인이 직접 연결된 사람의 수를 세는 가장 기본적인 지표입니다.

내가 바로 아는 사람이 몇 명인지를 측정합니다. 값이 10이면 나는 10명과 직접 연결되어 있습니다. 단순하지만 "인맥의 양"을 보여주는 직관적인 지표입니다.

degree(v) = 노드 v에 연결된 엣지의 개수
바깥 노드를 클릭해 연결 토글degree = 4
바깥 노드를 클릭하면 중심 노드(나)와의 연결이 추가/제거됩니다.
Metric 02Betweenness · Bridge

매개 중심성

서로 모르는 두 사람을 연결하는 '다리(bridge)' 역할의 정도입니다.

전체 네트워크에서 임의의 두 사람을 잇는 최단 경로 중, 나를 거쳐 가는 비율을 모두 합한 값입니다. 값이 높을수록 정보·자원이 흐르는 핵심 통로이며, 내가 빠지면 단절되는 관계가 많아집니다. 조직의 '브로커' 또는 '게이트키퍼'입니다. 전역(global) 지표로, 네트워크 전체를 알아야 계산됩니다.

betweenness(v) = Σ (v를 지나는 최단경로 / 전체 최단경로)
노드를 클릭하면 매개 중심성 값 표시노드를 선택하세요
노드를 클릭하면 그 사람이 지나는 최단 경로들이 강조되고, 매개 중심성 값이 표시됩니다. 두 그룹을 잇는 다리 노드의 값이 가장 큽니다.
조직 예시 — 두 부서를 잇는 유일한 연결점인 직원이 퇴사하면, 두 부서 간 정보 교류가 완전히 끊길 수 있습니다. 이 직원의 매개 중심성이 가장 높습니다.
Metric 03Closeness · Reach Speed

근접 중심성

네트워크 내 모든 사람에게 얼마나 빨리 도달할 수 있는지를 나타냅니다.

정보를 전달할 때 그룹 내 모든 사람에게 전파되는 속도입니다. 다른 모든 노드까지의 최단 경로 거리 합이 작을수록 근접 중심성이 높습니다.

closeness(v) = (N−1) / Σ 최단거리(v, 모든 노드)
노드를 클릭해 전파 단계 확인노드를 선택하세요
색이 진할수록 가깝고(1단계), 옅을수록 멀리(2~3단계) 떨어져 있습니다.
Metric 04Clustering · Closure

클러스터링 계수

내 지인들끼리 서로 얼마나 잘 아는지를 나타내는 값(0~1)입니다.

나(녹색 노드)는 옆에 두고, 나와 연결된 이웃들끼리의 연결만 봅니다. 이웃이 4명이면 이웃 간 가능한 연결은 6쌍입니다. 이 중 실제로 몇 쌍이 연결되어 있는지가 계수입니다. 모두 연결되면 c=1, 절반이면 c=0.5, 하나도 없으면 c=0. 값이 높을수록 촘촘한 소집단(클리크)이라 신뢰가 높지만 외부 정보 유입은 제한됩니다.

CC = 이웃들 간 실제 연결 수 / 가능한 모든 연결 수 [n(n−1)/2]
이웃들 사이 점선을 클릭해 연결/해제CC = 0.33
왼쪽 녹색 노드가 '나'입니다. 오른쪽 이웃 4명 사이의 점선을 클릭해 연결해 보세요. 이웃끼리 많이 연결될수록 계수가 1에 가까워집니다.
Metric 05Density · Team-level

네트워크 밀도

전체 네트워크가 얼마나 촘촘하게 연결되어 있는지를 0~1로 표현합니다.

가능한 모든 연결 중 실제로 연결된 비율입니다. 밀도가 높으면 신뢰·협력이 강하지만 새 정보가 적고, 낮으면 다양한 정보에 접근하기 좋지만 응집력이 약합니다.

density = 실제 엣지 수 / [N(N−1)/2]
슬라이더로 밀도 조절density = 0.30
희소촘촘
슬라이더를 오른쪽으로 옮길수록 더 많은 연결이 추가됩니다.
Metric 06Structural Hole · Burt 1992

구조적 공백

내 이웃들이 서로 얼마나 안 겹치는지를 봅니다. 서로 모르는 이웃들 사이의 '빈틈(공백)'을 메우는 사람이 정보·기회·통제력의 이점을 얻습니다.

Burt(1992)는 두 사람 사이에 직접 연결이 없는 빈 자리를 구조적 공백이라 불렀습니다. 내 이웃 A와 B가 서로 모른다면, 나는 A·B 사이의 공백을 잇는 유일한 다리이고 양쪽의 서로 다른 정보를 독점적으로 받습니다. 반대로 A·B가 서로 알면 그들의 정보는 중복(redundant)되고, 나를 거치지 않아도 연결되므로 내 중개 가치는 사라집니다.

대표 측정값은 두 가지입니다. ① 유효 규모(Effective size) = n − 2t/n — 내 이웃 n명 중 '중복을 걷어낸 실질 인맥 수'. 이웃끼리 연결(t)이 많을수록 작아집니다. ② 효율성(Efficiency) = 유효 규모 / n — 인맥 1명당 평균 이점. 1에 가까울수록 군더더기 없이 모두 비중복 정보원입니다.
유효 규모 = n − 2t/n · 효율성 = (n − 2t/n) / n
n = 이웃 수, t = 이웃들 사이의 실제 연결 수
이웃들 사이 선을 클릭해 연결을 추가/제거효율성 = 1.00
왼쪽 노드가 '나'입니다. 이웃 4명이 서로 모르면(연결 0개) 효율성 1.0 — 4명 모두 독립된 정보원. 이웃끼리 연결할수록 정보가 중복되어 유효 규모·효율성이 함께 떨어집니다. 빨간 점선이 아직 비어 있는 '구조적 공백'입니다.

또 다른 핵심 지표 — 네트워크 제약(Network Constraint)

Burt는 구조적 공백을 누리지 못하는 정도를 제약(constraint)으로 측정했습니다. 내 인맥들이 자기들끼리 똘똘 뭉쳐 있을수록(클리크) 나는 그 집단에 '갇혀' 새로운 정보·기회로 빠져나가기 어렵습니다 — 제약이 높습니다. 반대로 서로 모르는 여러 집단에 두루 걸쳐 있으면 제약이 낮고, 그만큼 자유롭게 중개·교섭할 수 있습니다.

제약 cᵢ = Σⱼ ( pᵢⱼ + Σ_q pᵢ_q · p_qⱼ )²
pᵢⱼ = i가 j에 투자한 관계 비중 · q = i,j 모두와 연결된 제3자
서로 다른 그룹(골드)을 잇는 빨간 선을 클릭해 연결/해제제약 = 낮음
녹색 '나'는 서로 분리된 4개 그룹에 연결되어 구조적 공백을 가로지르는 위치입니다(제약 최저). 두 그룹을 잇는 점선을 클릭해 연결하면 — 그 두 그룹이 더 이상 독립된 정보원이 아니게 되어(공백이 메워짐) 나의 제약이 올라가고 유효 규모가 줄어듭니다. 모든 그룹이 서로 연결되면 나는 더 이상 중개자가 아닙니다. 클릭하는 빨간 선이 바로 '메워진 공백'입니다.
3가지 지표 정리 — 유효 규모 = 중복 걷어낸 실질 인맥 수(클수록 좋음) · 효율성 = 인맥당 이점(1에 가까울수록 좋음) · 제약 = 공백을 못 누리는 정도(낮을수록 좋음). 셋 다 "내 이웃들이 서로 얼마나 안 겹치는가"라는 같은 직관을 다른 방식으로 잰 것입니다.
매개 중심성과 차이 — 매개 중심성은 전체 네트워크에서 내가 최단 경로 위에 얼마나 자주 놓이는지(전역·global)를 보고, 구조적 공백은 내 직접 이웃들이 서로 얼마나 안 겹치는지(지역·local)만 봅니다. 또 클러스터링 계수와는 정반대입니다: CC가 높으면(이웃끼리 다 앎) 구조적 공백은 사라집니다.
Metric 07Tie Strength · Granovetter 1973

유대 강도

관계의 강도는 함께한 시간·정서적 강도·친밀감·호혜성의 조합입니다.

Granovetter(1973)는 지인 소개로 취업한 사람들을 조사해, 정보의 대부분이 자주 만나지 않는 약한 유대로부터 왔음을 발견했습니다 — "사람들은 존재조차 잊고 있던 이들로부터 중요한 정보를 받는다."

취업 정보의 출처 (Granovetter 1973)
약한 유대(중간 빈도 + 연 1회 이하)가 전체 취업 정보의 약 83%를 차지했습니다.

약한 유대 (Weak Tie)

  • 다른 사회 그룹에 연결 → 새 정보·기회
  • 이질적 팀 → 다양한 관점·아이디어
  • 명시적·코드화 지식 탐색에 유리

강한 유대 (Strong Tie)

  • 높은 신뢰·응집력 → 깊은 협력
  • 동질적 팀 → 원활한 지식 공유
  • 암묵적·복잡한 지식 전달에 유리
Hansen 1999
